# きろくま 初期Issue一覧

## 運用方針

- Issueは小さく分割する
- 1 Issue = 1 PRを基本にする
- CodexにはIssue単位で実装を依頼する
- 完了条件にlint/test/buildを含める
- 対象外を明記し、スコープ膨張を防ぐ
- 実装前に関連docsとAGENTS.mdを読む
