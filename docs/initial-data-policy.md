# 初期データ投入方針

Issue #3時点では、DBスキーマとRepositoryの雛形のみを用意し、画面起動時の自動投入はまだ行わない。

## 方針

- 初期トレーニングメニューは Issue #6 で定義する。
- 初期データ投入処理は UI から直接呼ばず、usecase層または初期化専用モジュールからRepository経由で実行する。
- Dexieの `upgrade` はスキーマ移行に使い、通常のマスターデータ投入は冪等な初期化処理として分離する。
- 食材・レシピの初期データはMVPでは空で開始する。
- ユーザー入力データを上書きしないよう、固定IDまたは存在確認を使って重複投入を避ける。

## 対象候補

- workoutPlans
- workoutPlanExercises

その他のテーブルは、ユーザーが記録を追加して初めてデータが作られる想定とする。
